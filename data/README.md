# Data

## Public dataset candidates (requires Kaggle account to download)

Browse these and download any CSV that has: service dates OR odometer readings across multiple service events per vehicle.

| Priority | Dataset | URL | Notes |
|---|---|---|---|
| 1 | Car Predictive Maintenance | https://www.kaggle.com/datasets/pragyanaianddsschool/car-predictive-maintenance-data | Search snippets mention mileage, odometer, service history |
| 2 | Logistics Vehicle Maintenance History | https://www.kaggle.com/datasets/datasetengineer/logistics-vehicle-maintenance-history-dataset | Fleet historical records |
| 3 | Vehicle Service and Repair | https://www.kaggle.com/datasets/neerugattivikram/vehicle-service-and-repair-dataset-for-analysis | Service/repair focused |

### To download with Kaggle CLI:

```powershell
pip install kaggle
# Put your kaggle.json in C:\Users\<you>\.kaggle\kaggle.json
# (download from https://www.kaggle.com/settings → API → Create New Token)
kaggle datasets download -d pragyanaianddsschool/car-predictive-maintenance-data -p data/raw --unzip
```

Place any downloaded CSV in `data/raw/`. The training pipeline will use it if present; otherwise it falls back to the synthetic dataset.

---

## Synthetic dataset (fallback, always available)

Generated by `training/synthesize.py`. Simulates realistic driver profiles with noise around the standard 6-month / 10,000-km service thresholds.

```powershell
python -m training.synthesize
```

Output: `data/processed/service_history.csv`

### Schema

| Column | Type | Description |
|---|---|---|
| `months_driven` | float | Months since last service |
| `total_kms_driven` | float | Kilometres driven since last service |
| `days_until_service` | int | Target: days until service was actually due |
| `kms_until_service` | float | Target: km remaining until service was due |
| `driver_profile` | str | light / average / heavy (for EDA only, not an input feature) |
